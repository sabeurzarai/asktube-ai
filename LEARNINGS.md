# Learnings

- **2026-07-04** — `frontend/lib/analytics.ts` must NOT import from `@/lib/api`: the voice test suites (`__tests__/floating-companion-voice.test.tsx`, `__tests__/search-console-voice.test.tsx`) stub the whole module with `vi.mock("@/lib/api", () => ({...}))`, so any extra export pulled in by `analytics.ts` resolves to `undefined` and crashes `trackAnalyticsEvent` inside components. Keep the API-base fallback logic duplicated but in sync between `resolveApiBase` (api.ts) and `resolveAnalyticsApiBase` (analytics.ts).
- **2026-07-04** — In docker-compose, keys under `environment:` override values from `env_file:`. `CORS_ORIGINS` is therefore interpolated (`${CORS_ORIGINS:-...}`) in `docker-compose.yml` so a production `.env` value actually reaches the backend container.
- **2026-07-04** — The live EC2 demo maps the frontend to host port `3001` via a local override that is NOT in the repo; the repo's `docker-compose.yml` maps `3000:3000`. The Nginx snippet in DEPLOY.md proxies to `3001` — deploying from a clean checkout without adjusting one side yields 502 Bad Gateway (note added to DEPLOY.md §"Current live demo setup").
- **2026-07-04** — On Windows, `greenlet` fails to import (`DLL load failed ... filename or extension is too long`) when the venv lives under a very long path. Backend tests run fine from a venv at a short path (e.g. `C:\Users\sisaz\AppData\Local\Temp\at_venv`); only `pydantic`, `pydantic-settings`, `sqlalchemy`, `aiosqlite`, `greenlet`, `pytest`, `pytest-asyncio` are needed for `tests/test_analytics_service.py`.
