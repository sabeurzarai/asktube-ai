What Matches Carlos's Whiteboard

  • Primary transcript path uses youtube-transcript-api.
  • The normal flow avoids full YouTube video download.
  • No YouTube account authentication is used.
  • Audio download through yt-dlp is fallback only for Whisper transcription.
  • No copyrighted audio or video is committed to GitHub.
  • The RAG system answers from transcript evidence, not general internet knowledge.

Main Difference

Carlos suggested Gradio or Streamlit because they are fast choices for ML demos. AskTube AI uses Next.js because the goal became a more complete product-style learning platform.

"I used Next.js instead of Gradio/Streamlit to build a richer learning interface, but the required AI architecture is still fully implemented."

How the System Works

1

Search

User searches YouTube by text or voice, optionally filtering by video length.

2

Select

The frontend shows a cinematic carousel and the user chooses a video.

3

Transcript

The backend extracts public captions with youtube-transcript-api.

4

Vectorize

Transcript chunks are embedded and stored in ChromaDB with timestamps.

5

Retrieve

LangChain retrieves the most relevant transcript chunks for each question.

6

Answer

The chatbot answers with timestamp citations and keeps session memory.

Strong Points

  • Clear LangChain tool-calling architecture.
  • Transcript-only anti-hallucination rule.
  • Timestamp citations required.
  • ChromaDB vector retrieval.
  • Modern UI and strong UX polish.
  • Analytics dashboard for product, RAG, pipeline, UX, and business metrics.

Risk To Explain

YouTube often blocks transcript access from AWS and other cloud IP ranges. This is an infrastructure limitation, not a broken RAG pipeline.

Best demo strategy: use EC2 to prove deployment, and local Docker to prove full transcript ingestion if the proxy is blocked.

Final Verdict

AskTube AI does not only meet the minimum. It is technically above the expected level because it includes tools, memory, RAG, vector storage, evaluation, deployment, voice input, TTS, analytics, observability, and a polished product frontend.
